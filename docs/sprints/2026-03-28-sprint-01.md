## Hard Constraints

- [ ] **No display-only stubs** — every tool must call real implementation logic
- [ ] **All error states handled** — EmbeddingService fail state, empty index, zero results, malformed IDs
- [ ] **No hardcoded paths** — model cache in `~/.contextlens-turbo/models/`, db in `~/.contextlens-turbo/`
- [ ] **Security** — no eval, no shell exec, no user-controlled paths outside project root
- [ ] **TypeScript strict mode** — `strict: true`, no `any` without justification comment
- [ ] **ESM modules** — all imports use `.js` extensions, `"type": "module"` in package.json

---

## Project-Specific Constraints

- [ ] **MCP protocol compliance** — tool schemas must match MCP spec; all responses in `{ content: [{ type: "text", text: string }] }` format
- [ ] **Zero-dependency compression** — PolarQuant and QJL must be pure TypeScript with no external math libraries
- [ ] **Projection matrix determinism** — xorshift128+ seed=42 PRNG; identical matrix across all installs
- [ ] **Graceful fallback** — if `@huggingface/transformers` fails to load, all 7 tools continue working via FTS5
- [ ] **Bit packing correctness** — 3-bit angle packing and 1-bit QJL packing must handle byte-boundary crossover correctly
- [ ] **Schema additive** — v1→v2 migration never drops existing tables or rows
